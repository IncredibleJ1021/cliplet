# v0.4.1 Rollout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the tested reliability changes as cliplet v0.4.1 on GitHub Releases and Homebrew, and publish the npm installer when registry credentials are available.

**Architecture:** Treat the exact main-branch commit as the release unit. Push it, require CI success for that SHA, create the tag through the guarded script, verify the resulting artifacts, then update downstream distribution metadata from the immutable release zip.

**Tech Stack:** Git, GitHub Actions, GitHub CLI, Homebrew Cask, npm, shasum, hdiutil, cliplet release scripts.

## Global Constraints

- Version is exactly `0.4.1`; Git tag is exactly `v0.4.1`.
- Never move or recreate an existing release tag.
- Do not tag unless CI succeeded for the exact commit being tagged.
- GitHub Release and Homebrew Cask are required release outputs.
- npm publication occurs only if `npm whoami` succeeds and version 0.4.1 is not already published.
- Developer ID signing and notarization are not claimed in this release.

## File Structure

- Modify `CHANGELOG.md`: document user-visible v0.4.1 behavior.
- Modify `package.json`: set npm installer version to 0.4.1.
- Modify `npm/cliplet.js`: update displayed version example.
- Modify `README.md`: update release and installer examples to 0.4.1.
- Modify `Casks/cliplet.rb` only after the immutable release zip exists: set 0.4.1 and its exact SHA-256.
- No source file is changed during rollout unless CI or release verification exposes a defect; such a defect requires a normal fix commit and a fresh CI cycle before tagging.

### Task 5: Verify Assets and Update Homebrew Cask

**Files:**
- Modify: `Casks/cliplet.rb`
- External state: `origin/main`

**Interfaces:**
- Consumes: immutable v0.4.1 release zip
- Produces: Homebrew Cask version 0.4.1 with matching SHA-256

- [ ] **Step 1: Download and verify release artifacts**

```bash
RELEASE_DIR="/tmp/cliplet-v0.4.1-release"
rm -rf "${RELEASE_DIR}"
mkdir -p "${RELEASE_DIR}"
gh release download v0.4.1 --dir "${RELEASE_DIR}"
ZIP_PATH="${RELEASE_DIR}/cliplet-macOS-v0.4.1.zip"
DMG_PATH="${RELEASE_DIR}/cliplet-macOS-v0.4.1.dmg"
test -f "${ZIP_PATH}"
test -f "${DMG_PATH}"
hdiutil verify "${DMG_PATH}"
ditto -x -k "${ZIP_PATH}" "${RELEASE_DIR}/unzipped"
./scripts/verify_app.sh "${RELEASE_DIR}/unzipped/cliplet.app" 0.4.1
CASK_SHA256="$(shasum -a 256 "${ZIP_PATH}" | awk '{print $1}')"
test "${#CASK_SHA256}" -eq 64
printf '%s\n' "${CASK_SHA256}"
```

Expected: both assets exist; DMG and app checks pass; one 64-character hash is printed.

- [ ] **Step 2: Patch Cask with the literal verified hash**

Use `apply_patch` to change only the version declaration and SHA-256 declaration in `Casks/cliplet.rb`. Copy the literal 64-character value printed by Step 1 into the `sha256` string; do not retain the v0.4.0 hash.

```ruby
version "0.4.1"
```

- [ ] **Step 3: Verify the Cask against the release zip**

```bash
RELEASE_DIR="/tmp/cliplet-v0.4.1-release"
ZIP_PATH="${RELEASE_DIR}/cliplet-macOS-v0.4.1.zip"
CASK_SHA256="$(shasum -a 256 "${ZIP_PATH}" | awk '{print $1}')"
CASK_VERSION="$(ruby -ne 'puts $1 if $_ =~ /version "([^"]+)"/' Casks/cliplet.rb)"
CASK_HASH="$(ruby -ne 'puts $1 if $_ =~ /sha256 "([0-9a-f]+)"/' Casks/cliplet.rb)"
test "${CASK_VERSION}" = "0.4.1"
test "${CASK_HASH}" = "${CASK_SHA256}"
ruby -c Casks/cliplet.rb
```

Expected: all comparisons succeed and Ruby prints `Syntax OK`.

- [ ] **Step 4: Commit and push the Cask update**

```bash
git add Casks/cliplet.rb
git commit -m "Update Homebrew cask to v0.4.1"
git push origin main
```

Expected: the non-force push succeeds.

- [ ] **Step 5: Wait for post-release main CI**
